Setup Guide · Configurable Model

KOVA Schematics —
Implementation Process

Follow these steps in order when setting up a new configurable model for a KOVA project.

01

View Templates

The client requires two versions of the schematics: one with room tags and leader lines, and another with a clean drawing on the page. To support this, two view templates were created in the CDM and must be imported into the configurable model before proceeding.

View Templates

Schematic / KOVA Clean drawing, no annotations
Schematic / KOVA - Room Tags With room tags and leader lines

View Template Override

Schematic / KOVA Instance-level override
Note: All three items above must be imported from the CDM before proceeding to subsequent setup steps.
02

Sheet Templates

Sheet templates define how schematics are organized, named, sized, and annotated in the configurable model. Create new sheet templates under Sheets following the structure and conventions below.

Sheet organization

📁 Group: Schematics / KOVA
KOVA Schematics / Opt. Basement
KOVA Schematics / First Floor
KOVA Schematics / Opt. Second Floor
… one sheet per floor level

Naming convention: Group: Schematics / KOVA  +  Sheet: KOVA Schematics / [Floor]

Preferred sheet size

4'-6" W × 3'-8" H

Standard for all KOVA schematic sheets

Reference project

Davidson

IncludeGuidelines

ViewTemplate

View Template

Annotations and viewports

Name all viewports and annotations to reflect the content of the viewport
Group annotations according to the division of rooms and room groups
Copy textboxes from the Davidson project — it has all levels modeled and a considerable amount of options, making it the most complete available reference
Tip: Always set up base text annotations before start the viewports placement. As the viewports are added, update their names and after completing this steps, move forward with the addition of lines and leaders. This way, overall organization is easier to maintain as the sheet grows.
